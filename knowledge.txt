Problem Solving Approach:

I approach problems by first understanding the system instead of immediately jumping into implementation. I try to identify:
- What the actual problem is
- What assumptions exist
- What could fail at scale
- What tradeoffs are involved
- How maintainable the solution will be later

My usual approach:
1. Understand the problem deeply
2. Break it into smaller components
3. Identify constraints and edge cases
4. Build a simple working version first
5. Improve structure, scalability, and reliability gradually
6. Optimize only after correctness and clarity

When debugging:
- I isolate the failing component first
- Reproduce the issue consistently
- Validate assumptions using logs or smaller test cases
- Fix root causes instead of patching symptoms
